开源项目01 / 03
为什么需要它

问题不是 AI 不会写代码，是项目状态太容易散。

长线程一多，最先坏掉的往往不是代码，而是上下文、验收和下一步判断。

1
线程越长，背景越脏旧结论混进新判断，agent 开始顺着刚才的话讲。
2
做到哪一步，只能翻聊天phase、Done when、证据和 closeout 没有稳定落点。
3
它说完成，你一验收还有 bug缺少独立 verification 和 acceptance 的项目视角。
Threadsmith 的核心：不是让 AI 记住更多，而是让上下文更干净。
Context OS02 / 03
它怎么工作

把长线程，压成角色刚好够用的上下文。

T
Project Truth 目标、阶段、证据、验收写进 `.threadsmith/`。
C
Context Packet 把长聊天压成当前任务真正需要的信息。
R
Role Packet planner / executor / reviewer 各看各的。
S
Skill Orchestrator 发现本地 skills，缺失时回退到内置小协议。
